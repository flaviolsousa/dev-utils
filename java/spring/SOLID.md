SOLID Principles of OOP
Robert Uncle Bob Martin

```
S - Single Responsibility Principle
  --> Robustness and Cohesion
O - Open / Closed Principle
  --> OPEN to support more types requests and CLOSED for any modifications.
L - Liskov Substituition (Barbara Liskov)
  --> We can have many transparent implementations for a interface
I - Interface Segregation
  --> Slim interfaces
D - Dependency Inversion
  --> Coupling
```
